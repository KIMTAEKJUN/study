# 프로그래밍이란 ?
# 파이썬 외의 수백개의 언어가 존재하고, 모든 언어는 영어로 작성한다.
# 영문(작성) -> 컴파일러 -> 기계어 -> CPU -> 실행 -> 결과
# 간단히 설명 = 사람이 읽고 이해할 수 있는 언어를 가지고 어떤 프로그램을 만드는 과정을 프로그래밍이라 한다.

# 코딩 공부 방법 
# 1. 디버깅 -> 내 의도의 맞게 프로그램 실행 확인 ( 코딩 작성 중간중간에 디버깅하여 오류가 있는지 없는지 확인하며 개발한다. )
# 2. 알고리즘, -> 모든 기업에선 코딩 테스트가 필수, 구글에서 찾아 붙어넣기 하는것보단 훨 남, 프로그래밍 실력을 크게 향상 시킨다.
# 3. 다른사람 코드 참조 -> github -> 오픈소스 / 다른사람의 코드를 참조하는 것만큼 도움 되는 건 없고, 반드시 한번 살펴보길 바란다.
# 4. 주석 -> 다른 사람 또는 내가 남기고 간 코드를 다른 사람이 봤을 때 짧은 시간 안에 이해 할 수 있게 주석을 잘 다뤄야한다. 코드의 이해가 증가
# 5. 자기가 만들고 싶은 프로그램을 정확하게 정의 하는 것 -> 단위 단위 코딩을 만들며, 최종적으로 완성하는 게 제일 중요하다.

# 좋은 프로그램에서 나타나는 특징
# 1. 코드의 가독성 -> 시간과 비용을 낭비하지않도록, 처음부터 읽기 좋은 코드를 고려해서 다른 사람들이 봐도 이해 하기 쉽게 작성하는게 제일 중요하다.
# 2. 코드의 길이 -> 가독성과 연관
# 3. 변수의 이름 -> 코딩 테스트에서 변수 이름을 가지고 실력을 판단 할 수 있고, 어떤 의도로 이런 변수를 만들었구나라고 생각 할 수 있다.
# 4. 중복코드 줄이기 -> 불필요한 코드를 줄여야, 코드의 가독성이 좋아진다. 빠르게 실행되고 사양이 낮은 하드웨어에서도 잘 실행됀다. 코딩 기법에 따라 작성하는 것이 좋다.

# 빠른 개발속도 ( 생산성 ), 여러 사람과 협업이 수월 ( 매우 강력 )
